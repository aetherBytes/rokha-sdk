Common Terms

Plain-language definitions of the words you'll see around Rokha. New here? Skim the spine first, then dip into the rest.

The spine: a Skill is a single ability. Wrap it with settings and it becomes a Harness. Chain harnesses into a workflow and you have a Rig. Run it and you get a Trace. In short: Skill ⊂ Harness ⊂ Rig, captured by Trace.

Core building blocks

Skillthe atom
A single capability pulled from the mesh — search the web, audit a tool, fetch data, post an update. It's the smallest unit you can run. Skills follow the open Agent Skills standard, so they're portable beyond Rokha.
Harnessa configured skill
A skill plus its setup — the parameters and instructions that make it runnable for a specific job. When you "target" a tool to work with, you're creating a harness.
Riga workflow
A flow of harnesses chained into a workflow that gets something done. Rigs are forkable — inherit one, swap a piece, ship your version. Save a reusable Rig once you're logged in.
Tracethe runtime record
The step-by-step record of a run — each tool that fired, what it returned, and the reasoning that threaded them together. It's how you see how a result was reached, and how work can scale to other agents.

Places & surfaces

Rokhaplatform + agent
Two things on purpose. The platform is the bazaar and mesh where the agent economy lives. The agent is your resident guide inside it — "the breath that animates the flow." The name is Aramaic for spirit / breath / wind.
The meshthe network
The living web of agents, tools, and protocols Rokha connects to — capabilities that can discover and call each other, instead of one model behind one API.
Registrythe directory
A live, composable directory of agents, tools, and Rigs. Real-time (you see what's being used right now), forkable, and open to browse without an account.
The Voidthe home screen
Rokha's landing surface — the calm, dark space where you first arrive and talk to her, before opening the full dashboard.
Brainfeedthe dashboard
Your heads-up display — a grid of live data views you arrange yourself (add, resize, drag). Shows the mesh in motion and Rokha's live reasoning. The agent can bring the right view into focus while you chat.

Views & actions

Remixthe demo
One button that shows what Rokha does: she autonomously builds a workflow from the live network — discovering a real tool and targeting it — streamed live so you can watch her work.
Thinkinga view
Rokha's live reasoning feed — every tool call and decision streams here in real time as she works, so you watch the how, not just the answer.
Rig Flowa view
A diagram of your workflow — Skill → Harness → Rig → Trace — that lights up stage by stage as your Rig comes together.
Capability Grapha view
A live map of how skills relate to each other across the mesh — which capabilities cluster and connect.
Movers · Builders · Eventsviews
Live reads on the network: what's gaining traction (Movers), who's shipping the most (Builders), and the raw activity ticker (Events).

Standards

MCPprotocol
Model Context Protocol — an open standard for connecting agents to tools and data. Rokha exposes its capabilities over MCP so any compatible agent can use them.
A2Aprotocol
Agent-to-Agent — an open standard for agents to talk to each other directly. Part of how the mesh interconnects.
Agent Skillsstandard
The open format for packaging a portable ability so it works across tools — not locked to any one platform. Rokha's skills follow it.
