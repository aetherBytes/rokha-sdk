03

The cover — tell Rokha, she designs it

You never write HTML. That's not a missing feature — it's the security model: Rokha is the only author, and every cover renders inside a locked sandbox where scripts simply cannot run.

Open Profile → Page → Cover

Right under the Style panel.

Pick a style, describe what you want

Style chips get you started — clean & minimal, bold poster, terminal/hacker, soft editorial, playful — but the description is free-form. Cyberpunk neon with glitch flicker? Say exactly that.

Add imagery (optional)

Paste any public https image URL into the description, or just say "use my banner", "use my gallery", or "use my profile photo" — Rokha can read your own page's images.

Hit "✦ Ask Rokha to design it"

She designs the page in chat following her cover guidelines, saves it, and tells you to refresh. Want changes? Just keep talking to her — "more neon", "bigger title", "calmer". Say "reset my cover" any time to go back to the default.

No cover yet? You still look good — every page ships with a clean generated welcome built from your own name, tagline, accent, and real numbers.
04

What covers can and can't do

✓ Covers can

  • Any visual style — neon cyberpunk to letterpress minimal
  • CSS animation — pulses, scanlines, flicker, gradient motion
  • Images & fonts by https URL — hosted anywhere public
  • ASCII/Unicode art, bold typography, your accent color
  • Up to 64KB — a lot of room for HTML + CSS

✗ Covers can't

  • No JavaScript — refused on save AND inert in the sandbox
  • No working links or buttons — the cover is a poster; navigation lives in the page around it
  • No iframes, embeds, or forms
  • No file uploads yet — host images anywhere public and pass the URL
Why so strict? Your cover shows on a page other people visit. Double-walled safety — Rokha-only authoring + a zero-permission sandbox — means nobody's cover can ever run code in a visitor's browser. Yours included.
